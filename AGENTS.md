## Repo orientation

Next.js 16 App Router wiki. Content is markdown under `content/{recipes,wiki}/{en,es}/`. Client components live in `src/components/`, server pages in `src/app/[locale]/`. i18n via next-intl with strings in `messages/{en,es}.json`. Tests use vitest — use relative imports inside test files, the `@/` alias does not resolve in the test env.

## Pages

- `/` — landing with CTAs into each section
- `/ingredients` — A-Z grouped grid with category filter
- `/ingredients/[slug]` — wiki article rendered from `content/wiki/{locale}/{slug}.md`
- `/recipes` — A-Z grouped grid with meal-type filter
- `/recipes/[slug]` — recipe rendered from `content/recipes/{locale}/{slug}.md`
- `/meal-planner` — pick people + recipe count + meal type → suggest → shopping list. Tried recipes persist to localStorage, grouped by meal type with an undo control.
- `/sources` — the books grounding the wiki
- `/support` — donation pledge

## Content pipeline

Recipes and wiki entries are generated by the sibling repo `longevity-skills` at `~/Development/longevity-skills/`, not hand-written. The relevant skills:

- `/extract-book-knowledge` — book chapters → ingredient-claim JSON
- `/research-ingredient` — PubMed / Examine enrichment → enriched ingredient JSON
- `/generate-wiki-entry` — ingredient JSON → `content/wiki/{locale}/{slug}.md`
- `/generate-recipe` — 2-3 ingredient slugs → `content/recipes/{locale}/{slug}.md`

If editing content by hand, match the existing frontmatter schema (look at a neighbor file first).

## Locales

`en` is the source of truth. `es` is translated but lags — not every recipe or wiki entry has an `es` counterpart yet. When adding content, add the English file first; a Spanish version can follow in a separate pass.
